Project Proposal (Draft)

The PureCare Air Purifier project focuses on building an AI-powered air purification system that can recognize and adapt to user health conditions through audio-based symptom detection. Instead of reacting only to air quality metrics, the system learns from real-world cough and sneeze data to proactively adjust fan speed, purification level, and mode settings.

To achieve this, we are developing a robust sound classification model trained on diverse indoor recordings under realistic conditions. Each dataset sample represents different noise levels, distances, and body orientations to ensure adaptability to real household environments.

Ultimately, PureCare aims to enable a personalized, health-aware home environment—a purifier that not only improves air quality but also responds intelligently to each user’s physical condition.

Planned Methodology

  1. Audio Data Collection
    Following the protocol from Acoustic Cues for Person Identification using Cough Sounds (Tran et al.), each participant records 40 ten-second audio samples of coughs and sneezes across 5 separate sessions on different days. Recordings vary by:
    • Physical activity level before recording (rest, after walking, running, or climbing stairs)
    • Distance and position from the fixed device (0–3m, multiple room angles)
    • Body orientation relative to the microphone
    • Indoor background noise (quiet, appliances, music, or open windows)
    All recordings are performed indoors using hardware/tools/audio_capture.py with a stationary laptop simulating a typical purifier location.

  2. Metadata and Labeling
    Each sample is logged in a notes.csv file containing:
    user_id, file_name, day, sample_no, is_cough, physical_activity, bg_noise, distance, cough_type
    Example entry:
    M1US20-01, manual_save_time1761366192.csv, 0, 3, 1, rest, music+window, 2m, throat_clearing
    The dataset is designed to support both symptom detection and user identification research.

  3. AI Model Development
    • Sound Recognition: CNN-based classifier trained on MFCC features to distinguish cough, sneeze, and background noise.
    • Adaptive Control: Regression or reinforcement model that adjusts purifier behavior automatically when symptoms are detected.
    • User Identification: Feature embedding and clustering to associate new cough samples with individual users.

  4. Integration & Testing
    Deploy the trained models on a Raspberry Pi 4 using audio_capture.py for continuous local inference. Evaluate system accuracy, latency, and privacy-preserving performance in real indoor scenarios.

  5. Evaluation & Optimization
    Measure performance using detection accuracy, false positive rate, and energy efficiency. Refine models through iterative retraining with newly collected real-world data.
